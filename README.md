# CS 330 Portfolio Submission

## Project Overview

This repository contains my final project artifacts for **CS 330: Computational Graphics and Visualization**. For this project, I created a **3D desk scene in OpenGL** based on a real desk photo. My goal was to build a scene that looked recognizable and realistic while still using relatively simple geometry. Instead of relying on highly complex models, I built the scene from reusable primitive meshes and then used transformations, textures, materials, and lighting to make the environment feel more believable.

The final scene includes a desk, raised shelf, monitor, laptop stand with laptop, keyboard, mouse, mouse pad, pen cup with pens, coffee cup, and other desk accessories. Together, these artifacts demonstrate my ability to design, build, refine, and document a complete 3D graphics project.

## Reflection

### How do I approach designing software?

I approach designing software by first thinking about the overall goal and then breaking that goal into smaller, manageable parts. For this project, I started by deciding what kind of 3D scene I wanted to create and then identifying the objects that would make the scene feel complete and recognizable. Once I had that plan, I focused on how to build each object from simpler shapes instead of trying to create everything as one highly complex model. That approach helped me stay organized and made the project feel much more manageable.

This project helped me develop stronger design skills in visual composition, object breakdown, and scene planning. I had to think carefully about how objects should relate to each other in 3D space, how textures and materials would affect realism, and how lighting would change the final look of the scene. I also learned that good design is not only about the final appearance. It is also about building the project in a way that supports clarity, realism, and maintainability.

For my design process, I used a step-by-step approach. I began with the main desk structure and then added supporting objects one at a time. As I worked, I compared the scene to my reference image and adjusted scale, placement, and detail until the final result felt believable. In future work, I can apply this same design process by starting with the main structure first, then building outward in layers while making improvements through testing and revision.

### How do I approach developing programs?

I approach developing programs by building them in small pieces and testing as I go. That was especially important in this project because even small changes to transformations, textures, or lighting could affect the whole scene. Instead of trying to finish everything at once, I worked one object or one problem at a time and checked the results before moving on.

One new development strategy I used while working on this 3D scene was organizing the project into separate classes for major scene components instead of leaving everything in one large file. Breaking the project into parts such as the desk, monitor, desk objects, laptop stand, and pen cup made the code easier to read, debug, and improve. I also used reusable helper functions for transformations, textures, UV scaling, and materials, which made the code more maintainable and reduced repeated logic.

Iteration played a major role in my development process. Very little in this project was perfect on the first try. I had to adjust object sizes, positions, texture scaling, lighting, and scene organization multiple times before the final version looked right. Through the milestones, my approach to development became more structured and more intentional. Early on, I was mainly focused on getting objects on the screen. By the end of the project, I was thinking much more about modular design, code reuse, realism, maintainability, and how each decision affected the scene as a whole.

### How can computer science help me in reaching my goals?

Computer science helps me reach my goals by teaching me how to solve problems in a structured way and how to build systems that are both functional and organized. This project strengthened my ability to think logically, debug issues, and improve a program through iteration. Those are skills that apply far beyond graphics programming.

Computational graphics and visualizations gave me new knowledge that will help in my future educational pathway by improving my understanding of transformations, coordinate systems, textures, lighting, camera movement, and scene organization. It also helped me connect programming concepts to visual results, which made some abstract ideas feel more concrete. I believe that will help me in future courses where problem solving, code structure, and technical precision are important.

Professionally, this project gave me more experience with modular design, debugging, and building a polished final product over multiple milestones. Even if I do not directly work in graphics programming, the skills I used here still transfer well into software development in general. This project showed me how planning, iteration, and attention to detail can turn a basic program into something much more complete. It also gave me a stronger appreciation for how computer science can support work in areas like simulation, visualization, interactive applications, and game development.

## Closing Thoughts

This project helped me see that building a convincing 3D world is not just about drawing shapes. It also depends on making thoughtful decisions about design, structure, texture, lighting, object placement, and user interaction. Looking back, I can clearly see how much I improved from the early milestones to the completed project, and I am proud of the final result I was able to produce.
